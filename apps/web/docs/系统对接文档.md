# AI Academy 系统对接文档

本文档面向技术对接人员，说明如何将 AI Academy 与现有的用户系统和支付系统进行集成。

当前版本使用 **Supabase (PostgreSQL)** 云数据库存储数据。

| `courses` | 课程列表 |

## 二、数据模型

### 2.1 用户模型 (User)

```typescript
interface User {
  id: string;              // 用户唯一ID
  email: string;           // 邮箱
  password: string;        // 密码（对接后由后端处理）
  name: string;            // 用户名
  role: 'admin' | 'student';  // 角色
  permissions: string[];   // 已购买/注册的课程ID列表
  degreePermissions?: string[];  // 已购买的学位ID列表
}
```

### 2.2 课程模型 (Course)

```typescript
type CostType = 'free' | 'paid' | 'charity';

interface Resource {
  title: string;
  url: string;
  type: 'pdf' | 'code' | 'link';
}

interface Course {
  id: string;              // 课程唯一ID
  title: string;           // 课程标题
  description: string;     // 课程描述
  learningPoints: string[];  // 学习要点列表
  instructor: string;      // 讲师姓名
  level: 'Beginner' | 'Intermediate' | 'Advanced' | 'Expert';
  duration: string;        // 时长（如 "120 分钟"）
  videoDuration?: number;  // 视频时长（分钟）
  thumbnail: string;       // 封面图URL或Base64
  tags: string[];          // 标签列表
  costType: CostType;      // 付费类型
  price: number;           // 价格（人民币），免费为0
  videoUrl?: string;       // 视频URL
  resources?: Resource[];  // 课程资源列表
}
```

## 三、对接接口设计

Academy 前端需要调用以下 API，请后端提供对应接口。

## 四、前端对接改造

### 4.1 API 服务封装

创建 `services/api.ts`：

```typescript
const API_BASE = '/api/academy';

class ApiService {
  private token: string | null = null;

  setToken(token: string) {
    this.token = token;
    localStorage.setItem('ai_academy_token', token);
  }

  private async request(path: string, options: RequestInit = {}) {
    const headers: HeadersInit = {
      'Content-Type': 'application/json',
      ...options.headers,
    };

    if (this.token) {
      headers['Authorization'] = `Bearer ${this.token}`;
    }

    const response = await fetch(`${API_BASE}${path}`, {
      ...options,
      headers,
    });

    if (!response.ok) {
      throw new Error(`API Error: ${response.status}`);
    }

    return response.json();
  }

  // 认证
  async login(email: string, password: string) {
    return this.request('/auth/login', {
      method: 'POST',
      body: JSON.stringify({ email, password }),
    });
  }

  async register(email: string, password: string, name: string) {
    return this.request('/auth/register', {
      method: 'POST',
      body: JSON.stringify({ email, password, name }),
    });
  }

  // 课程
  async getCourses() {
    return this.request('/courses');
  }

  async getCourse(id: string) {
    return this.request(`/courses/${id}`);
  }

  // 注册免费课程
  async enrollCourse(courseId: string) {
    return this.request(`/enroll/course/${courseId}`, {
      method: 'POST',
    });
  }

  // 创建支付订单
  async createOrder(type: 'course' | 'degree', itemId: string) {
    return this.request('/orders/create', {
      method: 'POST',
      body: JSON.stringify({ type, itemId, paymentMethod: 'wechat' }),
    });
  }

  // 查询订单状态
  async getOrderStatus(orderId: string) {
    return this.request(`/orders/${orderId}/status`);
  }
}

export const api = new ApiService();
```

### 4.2 需要改造的组件

| 组件/功能 | 当前实现 (Supabase) | 对接主站后改造 |
|----------|---------|-----------|
| 登录/注册 | Supabase users 表 | 调用主站 `/auth/login`、`/auth/register` |
| 课程列表 | Supabase courses 表 | 可继续用 Supabase 或改为主站 API |
| 权限检查 | Supabase permissions 字段 | 调用主站 `/permissions/course/{id}` |
| 购买流程 | 显示微信二维码 | 调用主站支付接口，轮询订单状态 |
| 管理员授权 | Supabase 直接更新 | 调用主站 `/admin/grant-permission` |

### 4.3 支付流程改造

```typescript
// 购买课程/学位
const handlePurchase = async (type: 'course' | 'degree', itemId: string) => {
  try {
    // 1. 创建订单
    const { data } = await api.createOrder(type, itemId);

    // 2. 显示支付二维码
    showPaymentModal(data.qrCode, data.orderId);

    // 3. 轮询订单状态
    const pollInterval = setInterval(async () => {
      const status = await api.getOrderStatus(data.orderId);

      if (status.data.status === 'paid') {
        clearInterval(pollInterval);
        // 4. 刷新用户权限
        await refreshUserPermissions();
        showSuccessMessage('购买成功！');
      } else if (status.data.status === 'failed' || status.data.status === 'expired') {
        clearInterval(pollInterval);
        showErrorMessage('支付失败或已过期');
      }
    }, 3000); // 每3秒查询一次

  } catch (error) {
    showErrorMessage('创建订单失败');
  }
};
```

## 五、SSO 单点登录对接

如果主站已有用户系统，可以使用 SSO 对接。

### 5.1 方式一：Token 传递

主站登录后，将 token 传递给 Academy：

```
https://ai-academy.local?token=xxx
```

Academy 前端读取 token 并验证：

```typescript
useEffect(() => {
  const params = new URLSearchParams(window.location.search);
  const token = params.get('token');
  if (token) {
    api.setToken(token);
    // 获取用户信息
    api.getMe().then(user => setCurrentUser(user));
  }
}, []);
```

### 5.2 方式二：共享 Cookie

如果 Academy 部署在同一主域下（如 `ai-academy.local` 和 `ai-academy.local`），可以共享 Cookie。

后端设置 Cookie 时指定 domain：

```
Set-Cookie: token=xxx; Domain=.ai-academy.local; Path=/; Secure; HttpOnly
```

### 5.3 方式三：OAuth 2.0

使用标准 OAuth 2.0 流程，Academy 作为 Client 对接主站的 OAuth Server。

## 六、部署集成

### 6.1 子域名部署

推荐方式：将 Academy 部署到子域名。

```
主站: ai-academy.local
Academy: ai-academy.local
```

### 6.2 子路径部署

如需部署到子路径（如 `/academy/`），修改 Vite 配置：

```typescript
// vite.config.ts
export default defineConfig({
  base: '/academy/',
});
```

### 6.3 API 代理配置

Nginx 反向代理配置：

```nginx
# Academy 前端
location /academy/ {
    alias /var/www/academy/dist/;
    try_files $uri $uri/ /academy/index.html;
}

# Academy API 代理
location /api/academy/ {
    proxy_pass http://backend-server:8080/api/academy/;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
}
```
