# 实践项目功能开发文档

## 📝 功能概述

在课程详情页新增**实践项目**功能，让学员通过实际操作巩固所学知识，并与 AI Academy.com 的开源产品深度结合。

---

## 🗄️ 数据库设计

### 新增表结构

#### 1. `practice_projects` - 实践项目表

| 字段 | 类型 | 说明 |
|------|------|------|
| id | VARCHAR(191) | 主键 |
| course_id | VARCHAR(191) | 关联课程ID |
| title | VARCHAR(191) | 项目标题 |
| description | TEXT | 项目描述 |
| project_url | VARCHAR(191) | AI Academy 项目链接 |
| thumbnail_url | VARCHAR(191) | 项目封面（可选） |
| difficulty | ENUM | 难度：beginner/intermediate/advanced/expert |
| estimated_time | INT | 预计完成时间（分钟） |
| tags | TEXT | 标签（逗号分隔） |
| project_type | ENUM | 项目类型（见下方） |
| order_index | INT | 排序索引 |
| requirements | TEXT | 前置要求 |
| objectives | TEXT | 学习目标 |
| is_active | BOOLEAN | 是否激活 |
| created_at | DATETIME | 创建时间 |
| updated_at | DATETIME | 更新时间 |

**项目类型（project_type）：**
- `model_deployment` - 模型部署
- `model_training` - 模型训练
- `model_inference` - 模型推理
- `api_integration` - API集成
- `notebook` - Jupyter Notebook
- `sandbox` - 在线沙箱
- `repository` - 代码仓库
- `csghub_space` - CSGHub Space

#### 2. `practice_completions` - 实践完成记录表

| 字段 | 类型 | 说明 |
|------|------|------|
| id | VARCHAR(191) | 主键 |
| user_id | VARCHAR(191) | 用户ID |
| project_id | VARCHAR(191) | 项目ID |
| status | ENUM | 状态：in_progress/completed/skipped |
| started_at | DATETIME | 开始时间 |
| completed_at | DATETIME | 完成时间 |
| submission_url | VARCHAR(191) | 提交链接（可选） |
| notes | TEXT | 学习笔记（可选） |

**唯一索引：** (user_id, project_id) - 每个用户对每个项目只有一条记录

## 🔌 后端 API

### 路由前缀：`/api/v1/practices`

### 1. 获取课程的实践项目列表
```
GET /practices/courses/:courseId
```
**权限：** 公开
**返回：** PracticeProject[]

### 2. 获取实践项目详情
```
GET /practices/:id
```
**权限：** 公开
**返回：** PracticeProject

### 3. 创建实践项目
```
POST /practices
```
**权限：** 管理员
**请求体：** CreatePracticeProjectDto
**返回：** PracticeProject

### 4. 更新实践项目
```
PATCH /practices/:id
```
**权限：** 管理员
**请求体：** UpdatePracticeProjectDto
**返回：** PracticeProject

### 5. 删除实践项目
```
DELETE /practices/:id
```
**权限：** 管理员
**返回：** { message: string }

### 6. 获取用户实践进度
```
GET /practices/user/progress?courseId=xxx
```
**权限：** 需登录
**查询参数：** courseId（可选）
**返回：** PracticeCompletion[]

### 7. 开始实践项目
```
POST /practices/:id/start
```
**权限：** 需登录
**返回：** PracticeCompletion

### 8. 完成实践项目
```
POST /practices/:id/complete
```
**权限：** 需登录
**请求体：**
```json
{
  "submissionUrl": "https://github.com/user/repo",
  "notes": "学习心得..."
}
```
**返回：** PracticeCompletion

### 9. 跳过实践项目
```
POST /practices/:id/skip
```
**权限：** 需登录
**返回：** PracticeCompletion

## 🎨 前端实现

### 组件结构

```
apps/web/src/
├── lib/
│   └── practicesApi.ts          # API 客户端
├── features/
│   └── courses/
│       ├── CoursePracticesTab.tsx    # 实践项目 Tab
│       └── CourseDetailPage.tsx      # 课程详情页（需集成）
```

### 使用方式

在 `CourseDetailPage.tsx` 中添加实践项目 Tab：

```tsx
import { CoursePracticesTab } from './CoursePracticesTab';

// 在 Tabs 中添加
<Tabs>
  <Tab label="课程内容">...</Tab>
  <Tab label="学习资源">...</Tab>
  <Tab label="实践项目">
    <CoursePracticesTab />
  </Tab>
</Tabs>
```

### 功能特性

1. **项目列表展示**
   - 显示项目标题、描述、难度、预计时间
   - 标记项目类型和学习目标

2. **用户交互**
   - 开始实践：打开 AI Academy 项目链接，记录开始时间
   - 标记完成：提交学习链接和笔记（可选）
   - 跳过项目：标记为已跳过

3. **进度跟踪**
   - 显示已完成/进行中/未开始状态
   - 统计完成率

4. **权限控制**
   - 未登录用户点击"开始实践"跳转到登录页
   - 登录用户可以完整使用所有功能

## 🔍 常见问题

**Q: 如何对接 AI Academy.com 的验证机制？**

A: 目前采用用户手动标记完成的方式。如果 AI Academy 提供 API 回调或 Webhook，可以实现自动验证。需要在 `practice_completions` 表增加 `verified` 字段，并实现回调接口。

**Q: 如何防止学员虚假标记完成？**

A: 
1. 可以要求提交作品链接（GitHub、CSGHub Space 等）
2. 管理员可以查看学员提交的链接进行抽查
3. 后续可以集成 AI Academy API，自动验证用户是否真正完成了项目

**Q: 如何处理付费课程的实践项目？**

A: 实践项目跟随课程权限。只有购买了课程或拥有课程访问权限的用户才能看到和开始实践项目。验证逻辑在 `enrollments` 表中。

**Q: 实践项目可以跨课程复用吗？**

A: 当前设计是一个项目只能关联一个课程（course_id 为必填）。如果需要跨课程复用，可以创建多个相同内容但 ID 不同的项目记录，或者调整为多对多关系（新增中间表）。
