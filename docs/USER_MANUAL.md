## 1. 产品概览

OpenCSG Academy 是一个面向 **AI / 大模型工程师** 的在线教育与实践社区,核心由四个产品支柱构成:

| 支柱 | 是什么 | 主要受众 |
|------|--------|----------|
| **课程 (Courses)** | 单主题 1-10 小时的视频 + 文档 + 练习 | 所有人 |
| **学位路径 (Nano Degree)** | 4-8 门课的体系化学习路径(完成颁发学位) | 想系统入门/进阶的人 |
| **黑客松 (Hackathons)** | 限定时间的 AI 编程赛事,可组队可单人 | 想练手 + 拿奖的人 |
| **企业培训 (Enterprise)** | 团队定制化培训咨询 | 企业 HR / 培训负责人 |

**附加服务**:AI 助教(课程内问答)、积分 & 徽章、官方证书、订单系统、通知中心。

**产品定位**:不是「看完就完」的 MOOC,强调「**做出来才算**」— 每节视频课配实践作业,学位路径以作品集和项目交付为出口。

## 2. 快速开始

> 5 分钟从注册到第一节课

### 步骤 1 — 注册

1. 打开首页 https://academy.opencsg.com(开发环境: `http://localhost:5500`)
2. 右上角点 **「登录 / 注册」**
3. 选 **「立即注册」** 标签
4. 填邮箱 + 密码(至少 8 位,含字母数字)+ 昵称
5. 邮箱会收到验证邮件(部分场景免验证)
6. 验证后自动登录,跳回首页

### 步骤 2 — 浏览课程

首页向下滚,看到 **「热门课程」** 段,任选一张卡片点进去。

### 步骤 3 — 报名课程

- **免费课 / 公益课**:直接点 **「立即报名」**,0 跳到学习页
- **付费课**:点 **「立即购买」** → 跳订单页 → 支付 → 自动开通

### 步骤 4 — 开始学习

进入学习中心 (`/dashboard/learning`) 三栏布局:
- **左**:章节大纲
- **中**:视频 + 笔记 / 字幕 / 资源
- **右**:AI 助教

视频每 1 秒会向后端上报学习进度(开发模式下会看到 console log,正式环境无感知)。

### 步骤 5 — 完成课程

学完所有课时 → 系统自动生成 **「课程完成证书」** → 出现在 **「我的证书」** 页面。

## 3. 账号与登录

### 3.1 登录方式

OpenCSG Academy 支持 **邮箱 + 密码** 和 **第三方登录** 两种方式。

#### 邮箱 + 密码(当前主推)

- **注册**:邮箱 + 密码 + 昵称
- **忘记密码**:登录页点 **「忘记密码」** → 输邮箱 → 收到重置链接
- **密码规则**:≥ 8 位,必须含字母 + 数字(强密码要求)

#### 第三方登录(Phase 1 灰度中)

目前支持的 6 个第三方 provider:**Google / GitHub / 微信 / 企业微信 / 飞书 / Apple**

> ⚠️ **Phase 1 状态**:这 6 个入口均显示为灰色 + 锁定图标,鼠标悬停 tooltip 「即将推出,灰度开放中」。点任何按钮会弹 toast 提示。
>
> **Phase 2+**:开放 OIDC 接入,届时用户可一键绑定/解绑第三方账号。

### 3.2 账号绑定管理

登录后访问 **「设置 → 账号绑定」** (`/dashboard/settings/bindings`):

- 顶部:已绑定的第三方账号列表(Phase 1 暂只显示「本地账号」一项)
- 中部:6 宫格未绑定 provider 网格(灰度中)
- 底部:提示「至少保留一种登录方式」

### 3.3 认证技术细节(给好奇的人)

- **Access Token**:15 分钟有效,存内存(不写 localStorage 防 XSS 盗取)
- **Refresh Token**:7 天有效,后端写 **httpOnly Cookie** + `SameSite=Lax`
- **自动续期**:Access Token 过期时,前端自动用 Refresh Token 续,无需重新登录
- **退出**:清空内存 token + 后端销毁 refresh session

## 4. 首页浏览

首页(`/`)按 8 段位组织内容,从上到下:

| 段位 | 内容 | 关键操作 |
|------|------|----------|
| 1. **Hero** | 大标语 + 4 个平台数据点 + 2 个 CTA + 右侧课程预览 | 点 CTA 进课程列表 |
| 2. **热门课程** | 课程列表前 6 | 点卡片进详情 |
| 3. **学位路径** | 学位前 3 | 点卡片进学位详情 |
| 4. **黑客松** | 进行中的 3 个 + 倒计时 | 倒计时 0 自动跳详情 |
| 5. **AI 助教 CTA** | 品牌宣言 + mock 聊天气泡 | 体验 AI |
| 6. **讲师墙** | 4 个 mock 讲师 | 暂无交互(开发中) |

**响应式**:
- `≥ md` (768px+):FAB 在右下角

**加载失败 fallback**:任何 API 4xx / 5xx 时,自动降级到 mock 数据(4 课程 / 3 学位 / 3 黑客松 / 4 讲师),不显示空白页。

## 5. 课程系统

### 5.1 课程浏览

`/courses` — 课程列表,左侧筛选 + 右侧排序:

**左侧筛选**(可多选 + 清除全部):
- 分类:LLM 应用 / RAG / Agent / MLOps / 基础理论 / AI 产品
- 难度:Beginner / Intermediate / Advanced / Expert
- 时长:4 档(≤ 1h / 1-3h / 3-10h / 10h+)
- 收费:免费 / 付费 / 公益
- 标签:动态从结果里抽 top 8
- 讲师:动态从结果里抽
- 评分:4.0+ / 3.0+ / 全部

**右侧排序**:最新 / 最热门 / 评分(客户端排序,300ms debounce 拉数据)

**URL 同步**:`?q=搜索词&category=...&level=...&costType=...` 全部存 URL,刷新不丢、可分享。

**移动端**:筛选侧栏折叠成顶部一行(可点开 filter sheet)。

### 5.2 课程详情

`/courses/:id` — 关键信息:

- 顶部:封面 + 标题 + 讲师 + 时长 + 难度
- **学习要点** (5 条 markdown 列表)
- **价格卡片**:
  - 免费课:绿色徽章 + 「立即报名」
  - 付费课:价格 + 「立即购买」
  - 公益课:蓝徽章 + 「免费领」(内容同等付费,价格 0 全部捐赠公益项目)
- **章节大纲**(默认折叠前 3 章,点「展开」看全部)
- **试看章节**:第一课通常开放试看
- **相关评价**:滚动列表(分页 10 条/页)
- **相关推荐**:基于标签的 4 个相似课

### 5.3 报名流程

```
未登录 ──→ 注册/登录 ──→ 课程详情
                              ↓
                         立即报名(免费/公益)
                              ↓
                      写入 enrollments 表
                              ↓
                      解锁全部章节 + 跳学习中心
```

付费课多一步:报名 → 创建订单 → 跳支付 → 支付成功回调 → 开通。

### 5.4 学完一门课

- **完成条件**:视频课看完 + 实践作业提交(如该课有)
- **完成后**:
  1. 颁发 **「课程完成证书」**(自动,可下载)
  2. 累加 **积分**(基础 50 + 难度系数)
  3. 解锁 **课程专属徽章**(如有)
  4. 课程在 profile 显示 ✓

## 6. 学位路径 (Nano Degree)

### 6.1 学位是什么

Nano Degree 是一组「按学习顺序串联」的课程包,完成全部课程后颁发 **学位证书**。

**学位结构**:
- 4-8 门课(每门课内部独立报名)
- 1-2 个 capstone 项目(必须完成才能拿学位)
- 预估总学时 60-200 小时

### 6.2 学位列表

`/degrees` — 学位卡片网格:

每张卡片显示:
- 学位图标(emoji/符号)
- 标题 + 一句话描述
- 课程数 / 总章节 / 预估学时 / 在学人数
- 价格 chip(免费 / 付费 / 公益)

### 6.3 学位详情

`/degrees/:id` — 学位主页(也是学位「学习中心」):

- **Hero**:学位大图 + 标题 + 简介 + 「开始学习」CTA
- **学习路径**:3-8 个步骤(stage),每个步骤对应一门课,横向「进度条」串起来
  - ✓ 已完成(绿色)
  - ◐ 进行中(蓝色脉冲点)
  - ○ 未开始(灰)
- **Capstone 作品要求**:页面底部列出需要提交的作品说明
- **学位证书预览**:完成全部后,这里会显示学位证书的缩略图

### 6.4 完成学位

- 完成全部 4-8 门课 + 提交 capstone
- 系统自动 **「学位完成证书」**(比单课证书更有含金量)
- 学位 chip 在 profile 显示「🎓 学位 + 名称」

## 7. 黑客松 (Hackathon)

### 7.1 黑客松浏览

`/hackathons` — 列表,顶部 5 个状态 tab:

| 状态 | 含义 | 可操作 |
|------|------|--------|
| **全部** | 所有黑客松 | 看 + 报名(看状态) |
| **报名中 (upcoming)** | 还没开始,正在接受报名 | **可报名** |
| **进行中 (active)** | 比赛进行中 | 提交作品 |
| **评审中 (judging)** | 评审正在打分 | 等结果 |
| **已结束 (finished)** | 比赛结束,结果已出 | 看排名 / 领奖 |

每张卡显示:
- 比赛名 + 状态 chip(颜色编码:绿/蓝/橙/灰)
- 奖金池 / 奖品
- 开始 / 结束时间
- 队伍人数限制(min-max)
- 报名截止倒计时(仅 `upcoming` 状态)
- 标签(线上/线下/主题)

### 7.2 黑客松详情

`/hackathons/:id` — 比赛主页:

- **顶 Banner**:封面 + 比赛名 + 状态 + 「立即报名」CTA
- **基本信息**:时间 / 地点 / 参赛人数 / 奖金
- **比赛规则**:markdown 渲染
- **奖品**:分级奖金 + 赞助商奖品
- **时间线**:报名 → 开始 → 提交截止 → 评审 → 结果公布
- **Tab 切换**:
  - **公告**:组委会更新
  - **参赛团队**:已报名团队列表
  - **作品提交**:比赛进行中可提交
  - **评审标准**:评分维度

### 7.3 报名 / 组队

- **单人参赛**:点「立即报名」,组委自动分配或你自己创建团队
- **组队参赛**:点「创建团队」→ 邀请队友(通过用户名/邮箱)→ 队长提交报名
- **团队规模**:每场比赛有 min-max 限制(如 1-4 人)
- **退出团队**:开始前可自由退出,开始后需联系组委

### 7.4 提交作品

比赛进行中(进入 `active` 状态后),`/hackathons/:id/submit` 出现:

- 作品标题
- 仓库 URL(GitHub / GitLab)
- 在线演示 URL(可选)
- 视频 demo URL(可选)
- 简短描述(markdown)

提交后**锁定,不可修改**(避免最后时刻改). 最终提交需队长点「确认提交」。

### 7.5 评审与获奖

- 评审阶段 `judging` 状态期间,作品冻结
- 结果公布:`finished` 状态后,排名 + 奖品在详情页 + 个人通知
- 获奖团队 → 颁发 **「黑客松证书」**(带名次)

## 8. 学习中心 (Dashboard)

`/dashboard` 或 `/dashboard/learning` — P0-6 重设计的三栏布局:

### 8.1 三栏结构

```
┌─────────────────────────────────────────────────────┐
│  [返回] 课程名 · 进度条 X%        主题切换  学员信息 │
├──────────┬─────────────────────┬──────────────────┤
│          │                     │                  │
│  左:章节 │   中:视频 + tabs     │  右:AI 助教      │
│  (280px) │   (1fr)             │  (360px)         │
│          │                     │                  │
│  折叠大纲│  视频 / 笔记 / 字幕  │  4 问 1 答聊天   │
│  状态 ✓  │   / 资源            │  + 输入框        │
│          │                     │                  │
└──────────┴─────────────────────┴──────────────────┘
```

### 8.2 响应式

| 断点 | 布局 |
|------|------|
| `lg+` (≥1024px) | 三栏并排 |
| `md` (768-1023px) | 两栏(隐藏 AI 右栏,改 FAB 抽屉),AI 抽屉复用同一组件 |
| `< md` | 单栏 + 顶部 3 tab 切换(大纲 / 视频 / AI) |

### 8.3 视频 / Tabs

- **视频**:点章节大纲中的小节 → 中栏加载视频 + 自动记录进度
- **Tab 切换**:
  - **笔记**:本课个人笔记(markdown 编辑器,自动保存)
  - **字幕**:视频逐字稿,可点时间戳跳视频
  - **资源**:本课相关 PDF / 代码 / 链接
- **底部 sticky**:「完成本节」按钮,点了才记 ✓

### 8.4 AI 助教(右栏)

- **4 个常见问题 chips**(点击直接发问)
- **聊天流**:用户 + AI 交替气泡,AI 回答带「引用回到 Lesson X.X」超链接
- **免责声明**:**「AI 回答可能不准确,请以视频内容为准」** 灰色底,常驻底部

### 8.5 进度跟踪

- **上报机制**:每 1 秒 video time + 1,触发 `LearningEvent` 上报
- **后端记录**:`/api/v1/progress/lessons/:id/complete` (TODO 后续接)
- **当前状态**:`Development` 模式下,LearningEvent 走 console.log(标 `TODO(后端)`),生产模式正常
- **完成判定**:视频看完 + 「完成本节」按钮点了 → 100%

## 9. AI 助教

### 9.1 是什么

基于 LLM 的智能问答,知道当前课程内容,能给出 **带引用的回答**。

### 9.2 在哪用

- **学习中心右栏** (见 §8.4)
- **右下角 FAB 浮动按钮**(任意页面可点)
- **移动端**:FAB 在 bottom tab 上方,点开跳学习中心 AI 栏

### 9.3 怎么问

- 自由输入框:问任何与课程相关的问题
- 4 个快捷 chips:常见问题模板
- 快捷键:**⌘ + I**(待实现,目前只能点 FAB)

### 9.4 回答格式

每条 AI 回答的结构:

```
[回答正文]
─────────
📎 引用: Lesson 1.3 · 04:00  ← 可点跳回视频那个时间点
```

引用是 AI 自报「我从课程这个位置找的答案」,你点引用会跳视频到对应时间戳。

### 9.5 局限与免责

- **不要把 AI 当真理**。回答可能不准确 / 拼凑 / 上下文丢失
- **复杂推理**(数学、代码 review)优先看视频 + 自己跑代码
- **隐私**:不要输入个人隐私(密码、手机号等)

## 10. 积分与徽章

### 10.1 积分 (Points)

- **来源**:
  - 完成课时:基础 5 积分/节
  - 完成课程:基础 50 + 难度系数(Beginner ×1, Intermediate ×1.5, Advanced ×2, Expert ×3)
  - 完成学位:基础 500 + 课程数 ×100
  - 黑客松获奖:一等奖 500,二等奖 300,三等奖 100
- **去路**:目前**只读**,不消耗(Phase 2+ 可能开放积分商城)
- **查看**:`/dashboard/learning` 顶部学员信息 / 后续个人页

### 10.2 徽章 (Badges)

- **类型**:
  - 课程徽章:学完特定课程解锁
  - 学位徽章:完成学位解锁
  - 黑客松徽章:参赛/获奖解锁
  - 成就徽章:连续学习 N 天 / 累计 X 积分
- **展示**:`/dashboard` 学员信息区可见已解锁徽章
- **Phase 2+**:个人页有徽章墙,全网可见

## 11. 证书系统

### 11.1 证书类型

| 类型 | 触发条件 | 含金量 |
|------|----------|--------|
| **课程证书** | 完成单门课程 | ⭐ |
| **学位证书** | 完成 Nano Degree 全部课程 + capstone | ⭐⭐⭐ |
| **黑客松证书** | 黑客松完成 + 排名(参与/前三) | ⭐⭐(参与)/⭐⭐⭐(前三) |

### 11.2 查看证书

`/dashboard/certificates` — 4 tab:全部 / 课程 / 学位 / 黑客松

每张证书卡显示:
- 缩略图(渐变背景 + 证书编号)
- 标题 + 类型 chip
- 编号(如 `CSG-2026-COURSE-ABC123`)
- 颁发时间
- 操作:**查看 / 下载 / 验证**

### 11.3 公开验证

任何人(包括未登录用户)可以访问:
```
https://academy.opencsg.com/verify/<serial>
```

页面会显示:
- 证书持有者(脱敏:姓 + 名字首字母)
- 课程 / 学位 / 黑客松
- 颁发时间
- 防伪二维码

**用途**:求职时给 HR 看,或 LinkedIn 挂证书页。

### 11.4 证书图下载

- **复制验证链接**:点「复制验证链接」→ 公开验证 URL 进剪贴板,可贴到 LinkedIn / 邮件

## 12. 订单与支付

### 12.1 创建订单

付费课 / 付费学位 / 黑客松报名费,会先创建订单再跳支付。

### 12.2 订单状态

`/dashboard/orders` — 5 tab:

| 状态 | 含义 | 可操作 |
|------|------|--------|
| **待支付 (pending)** | 订单创建未支付 | **支付 / 取消** |
| **已支付 (paid)** | 支付成功 | 查看 / 申请退款 |
| **已取消 (cancelled)** | 用户主动取消 | - |
| **已退款 (refunded)** | 退款成功 | - |
| **已过期 (expired)** | 超过支付时限 | **重新支付** |
| **失败 (failed)** | 支付通道失败 | **重新支付** |

> 注:`cancelled` / `expired` / `failed` 3 种状态在 UI 归到「已取消」tab 展示(命名简化,数据是分开的)。

### 12.3 支付通道

- **Stripe**(国际信用卡):Visa / Master / Amex
- **支付宝**(国内):扫码 / 跳转
- **微信支付**(国内):扫码 / 公众号

支付回调成功后,订单自动置为 `paid` 并触发课程开通(可能 1-2 秒延迟,刷新页面即可)。

### 12.4 退款规则

后端已落地校验(见 `apps/api/src/modules/orders/orders.service.ts: refundOrder`)。规则:

- 课程订单:
  - 未开始(0 已完成 lesson):**全额退款**
  - 已开始 + 开通 ≤ 7 天 + 进度 < 20%:**95% 退款**(扣 5% 手续费)
  - 开通 > 7 天 **或** 进度 ≥ 20%:**不支持退款**
- 学位订单:
  - 所有关联课程都未开始 → **全额退**
  - 任一关联课程已开始 → **不支持退款**
- 黑客松:开赛 24h 前可退,之后不退

申请退款在订单详情页点 **「申请退款」** → 弹「确认申请退款?」二次确认 → 提交 → 1-3 工作日到账。返回的 `refundAmount` 字段反映实际退金额(已扣手续费)。

## 15. 搜索 (⌘K)

### 15.1 触发

- **macOS**:`⌘ + K` 或 `Ctrl + K`
- **Windows / Linux**:`Ctrl + K`
- **点击**:顶部导航的搜索图标

### 15.2 行为

- 弹层打开 → 焦点自动在搜索框
- 输入关键词 → **200ms debounce** → 实时拉取
- 结果按类型分组,每组最多 4 条:
  - 课程 (BookOpen 图标)
  - 学位 (GraduationCap 图标)
  - 黑客松 (Trophy 图标)
  - 讲师 (User 图标)
- **键盘导航**:
  - `↑ ↓` 在所有可见结果间切换(跨组)
  - `Enter` 跳当前选中
  - `⌘ + Enter` 新窗口打开
  - `Esc` 关闭弹层

### 15.3 响应式

| 断点 | 弹层形态 |
|------|----------|
| `< md` (移动端) | 全屏 modal |
| `md-lg` (tablet) | 居中 600×400 |
| `≥ lg` (desktop) | 居中 720×440 |

### 15.4 空状态

- 无输入:显示「热门搜索」4 chips
- 加载中:3 行 Skeleton
- 0 结果:`EmptyState` 「没找到相关内容,试试其他关键词」

### 15.5 搜索结果页

`/search?q=关键词` — 弹层外的全功能搜索页(分页 / 全部结果 / 高级筛选)。

## 18. 主题与无障碍

### 18.1 主题切换

- **位置**:顶部 nav 右侧 ☀️ / 🌙 按钮
- **行为**:
  - 切到 Dark → `<html class="dark">` + localStorage 持久化
  - 切到 Light → 移除 class
- **首次访问**:读 system 偏好(`prefers-color-scheme`)
- **持久化**:localStorage key `theme`

### 18.2 暗色模式

所有页面均支持暗色,使用语义化 token:
- `bg-neutral-50` / `bg-neutral-950`
- `text-neutral-900` / `text-neutral-50`
- `border-neutral-200`

第三方组件(Tailwind 写的) 100% 覆盖,React Query loading 状态也走 token。

### 18.3 无障碍 (A11y)

- **键盘导航**:全部交互元素 Tab 可达
- **焦点环**:可见,符合 WCAG 2.1 AA
- **ARIA 标签**:所有 icon-only 按钮带 `aria-label`
- **屏幕阅读器**:Toast 用 `aria-live="polite"`,加载态用 `role="status"`
- **颜色对比**:暗色 / 亮色 均 ≥ 4.5:1

## 20. 常见问题 (FAQ)

### Q1. 课程能下载离线看吗?

**A**:暂不支持(Phase 2+ 会做)。目前视频在线流式播放,需要稳定网络。

### Q2. 证书是否被企业认可?

**A**:OpenCSG 是国内头部开源大模型公司,旗下 Academy 证书在 **OpenCSG 生态企业**(模型厂商、Agent 平台、MLOps 公司)认可度较高。第三方企业认可度因公司而异,建议面试时主动出示 `/verify/<serial>` 验证页 + GitHub 项目作品集。

### Q3. 学完课能退款吗?

**A**:见 §12.4。简单说:**未开始 → 全退;开始 < 7 天 + 进度 < 20% → 95% 退;其他不退**。

### Q4. 黑客松可以跨队吗?

**A**:比赛开始后**不能跨队**,队员不能调整。开始前可自由加入退出。

### Q5. 学位可以单独买其中一门课吗?

**A**:可以。学位的每门课都是独立 SKU,可在课程详情页单独报名。但**只买部分课程拿不到学位证书**,需要完成全部 4-8 门。

### Q6. AI 助教支持多语言吗?

**A**:目前主要优化中文问答。英文可以问,但回答质量略差。Phase 2+ 会有 i18n 切换。

### Q7. 我能在多个设备登录吗?

**A**:可以,但同账号**最多 3 个设备同时在线**。超出会踢出最早登录的设备(不会丢数据)。

### Q8. 删除账号后证书还有吗?

**A**:**没有**。删除账号会清除所有数据(课程进度、订单、证书、积分)。如需保留证书,请先下载 PDF 留存。

### Q9. 课程难度跟我的水平不匹配怎么办?

**A**:点课程详情页的「难度」chip,看具体要求。**Beginner** 假设你有 Python 基础;**Intermediate** 假设你做过 1-2 个 LLM 项目;**Advanced** 假设你懂 Transformer 源码;**Expert** 假设你能独立训练模型。

### Q10. 我是教育机构,能批量采购吗?

**A**:可以,走 §14 企业培训通道,或邮件 enterprise@opencsg.com 直谈。

## 21. 故障排查

### 21.1 登录 / 账号

| 症状 | 排查 |
|------|------|
| 收不到验证邮件 | 1) 查垃圾邮件 2) 等 5 分钟 3) 没收到点「重新发送」 |
| 忘记密码链接过期 | 链接 30 分钟有效,重新申请 |
| 登录后被踢回登录页 | Refresh Token 过期(7 天),重新登录 |
| 第三方登录按钮灰 | Phase 1 灰度中(见 §3.1),等开放 |

### 21.2 视频 / 学习

| 症状 | 排查 |
|------|------|
| 视频卡顿 / 加载慢 | 1) 切到 720p(默认 1080p) 2) 检查网络 3) 刷新页面 |
| 进度不更新 | DevTools Console 看 `[LearningEvent TODO]` 日志(开发模式正常,生产模式不会出现) |
| 视频不播放 | 1) 浏览器禁用 Flash 旧插件 2) 试 Safari / Chrome 切换 |
| 字幕不显示 | 课程可能未提供字幕,联系 admin@opencsg.com 反馈 |

### 21.3 订单 / 支付

| 症状 | 排查 |
|------|------|
| 支付成功但课程未开通 | 等 1-2 分钟刷新;5 分钟未开通联系 support@opencsg.com 附订单号 |
| 看不到订单 | 登录账号是否正确;订单中心 `/dashboard/orders` 看 |
| 申请退款被拒 | 检查退款规则(§12.4),可邮件申诉 |

### 21.4 证书

| 症状 | 排查 |
|------|------|
| 课程完成没拿到证书 | 1) 检查所有课时 ✓ 2) 检查作业(如有)已提交 3) 等 5 分钟自动签发 |
| `/verify/:serial` 404 | 证书编号错误,回到证书详情页复制正确编号 |
| 证书图下载失败 | 改用其他浏览器 / 清缓存重试 |

### 21.5 性能

| 症状 | 排查 |
|------|------|
| 首页加载慢 | 1) DevTools Network 看哪个 chunk 大 2) 清浏览器缓存 3) 试无痕模式 |
| 搜索弹层卡顿 | 输入过快,200ms debounce 中,稍等 |
| 移动端发热 / 耗电 | 学习中心视频 1080p 是高耗电源,切 720p |

### 21.6 报错信息

| 报错 | 含义 | 解决 |
|------|------|------|
| `401 Unauthorized` | 登录态过期 | 重新登录 |
| `403 Forbidden` | 权限不足(比如非 admin 访问 admin) | 联系管理员 |
| `404 Not Found` | 资源不存在 | 检查 URL |
| `429 Too Many Requests` | 触发限流(全局 60 req/min) | 等 1 分钟 |
| `500 Internal Server Error` | 后端 bug | 报错截图发 support@opencsg.com |
