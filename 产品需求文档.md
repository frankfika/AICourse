# CourseAI 产品需求文档 (PRD)

> **部署方式**：本地部署，轻量化技术栈  
> **数据库**：SQLite（无需安装独立数据库服务）  
> **文件存储**：本地文件系统  
> **运行要求**：Node.js 18+ + macOS/Linux/Windows  
> **启动命令**：`pnpm install && pnpm dev`

---

## 一、项目概述

### 1.1 项目背景
CourseAI 是一个专注于 AI 课程的在线学习平台，旨在提供高质量的 AI 相关课程，并通过 Nano Degree 项目为学习者提供系统化的学习路径和认证。

### 1.2 项目目标
- 为学习者提供清晰、易用的课程浏览和学习体验
- 建立完整的课程到认证的学习路径（Nano Degree）
- 提供便捷的后台管理系统，支持课程和 Nano Degree 的维护

### 1.3 目标用户
- **学习者**：希望学习 AI 相关知识的个人用户
- **管理员**：负责课程内容和 Nano Degree 项目的维护人员

## 二、核心功能模块

### 2.1 前台展示系统

#### 2.1.1 Landing Page（首页）
**功能描述**：网站的入口页面，展示平台核心价值和内容概览

**核心元素**：
- **Banner 轮播区**：
  - 支持 3-5 张轮播图
  - 可配置跳转链接
  - 显示活动、新课程或重要公告
  
- **Hero Section**：主视觉区域，突出平台特色
  
- **课程分类导航**：
  - 展示主要课程分类（如：机器学习、深度学习、NLP、计算机视觉等）
  - 点击进入对应分类的课程列表
  
- **热门课程展示**：
  - 精选 3-6 门热门课程卡片
  - 显示浏览量或学习人数
  
- **Nano Degree 展示**：展示 2-4 个 Nano Degree 项目
  
- **平台特色说明**：简要介绍平台优势（用图标+文字形式）
  
- **数据展示**：
  - 课程总数
  - Nano Degree 总数
  - 学习者总数（如果有统计）
  
- **CTA 按钮**：引导用户浏览课程或 Nano Degree

**设计要点**：
- 简洁现代的设计风格
- 清晰的信息层级
- 流畅的导航体验
- 视觉吸引力强

#### 2.1.2 课程（Course）模块

##### 2.1.2.1 课程列表页
**功能描述**：展示所有可用课程的列表页面

**核心元素**：
- **面包屑导航**：首页 > 课程 > [分类名称]（如果有）

- **搜索栏**：
  - 关键词搜索
  - 实时搜索建议（可选）
  
- **筛选侧边栏/顶部筛选器**：
  - 按分类筛选（机器学习、深度学习、NLP 等）

- **排序功能**：
  - 最新发布
  - 最受欢迎（按浏览量）
  - 时长从短到长
  - 时长从长到短
  
- **课程卡片网格布局**（每个卡片包含）：
  - 课程封面图
  - 分类标签
  - 难度等级标签

  - 课程简介（简短描述，50-80字）
  - 课程时长

  - "查看详情"按钮

**交互逻辑**：
- 点击课程卡片进入课程详情页
- 筛选和排序实时更新列表
- 支持响应式布局
- 显示当前筛选条件和结果数量

##### 2.1.2.2 课程详情页
**功能描述**：展示单个课程的完整信息

**核心元素**：
- **面包屑导航**：首页 > 课程 > [分类] > 课程名称

- **课程头部区域**：
  - 课程标题
  - 分类标签 + 难度等级标签
  - 课程封面图或预告视频（可播放）
  - 关键信息卡片：
    - 课程时长
    - 难度等级
    - 更新时间
    - 浏览量/学习人数
    - 建议学习周期（如：2周，每周3小时）
  - 讲师信息（头像、姓名、职位）
  - "开始学习"主按钮
  - 分享按钮（分享到社交媒体）
  
- **Tab 导航区域**：
  - **Tab 1: 课程介绍**
    - 课程详细描述（支持富文本）
    - 你将学到什么（学习目标列表）
    - 适合人群
    - 先修要求/前置知识
    - 课程亮点
  
  - **Tab 2: 课程大纲**
    - 章节列表（可展开/收起）
    - 每个章节显示：
      - 章节标题
      - 章节时长
      - 章节主题（要点列表）
    - 显示总章节数和总时长
  
  - **Tab 3: 讲师介绍**
    - 讲师头像和基本信息
    - 讲师详细简介
    - 专业背景
    - 教学经验
    - 其他授课课程列表
  
  - **Tab 4: FAQ**
    - 常见问题解答
    - 可折叠的问答列表
  
  - **Tab 5: 相关推荐**
    - 相关课程推荐（同分类或同讲师）
    - 包含该课程的 Nano Degree（如果有）

- **侧边栏/底部固定栏**：
  - "开始学习"按钮（始终可见）
  - 课程关键信息快速查看
  
- **其他元素**：
  - 页面底部：更多课程推荐

**数据结构**：
```typescript
interface Course {
  id: string;
  title: string;
  slug: string;
  description: string; // 详细描述（支持富文本/Markdown）
  shortDescription: string; // 简短描述（50-80字）
  coverImage: string;
  previewVideoUrl?: string; // 预告视频 URL
  duration: number; // 总时长（分钟数）
  level: 'beginner' | 'intermediate' | 'advanced'; // 难度等级
  categoryId: string; // 课程分类 ID
  category: Category; // 课程分类
  instructor: Instructor;
  syllabus: Chapter[];
  prerequisites: string[]; // 先修要求
  learningObjectives: string[]; // 学习目标
  highlights: string[]; // 课程亮点
  targetAudience: string; // 适合人群
  suggestedWeeks?: number; // 建议学习周数
  hoursPerWeek?: number; // 每周建议学时
  faqs: FAQ[]; // 常见问题
  viewCount: number; // 浏览量
  enrollCount?: number; // 学习人数（可选）
  status: 'draft' | 'published' | 'archived'; // 状态
  featured: boolean; // 是否为热门课程
  createdAt: Date;
  updatedAt: Date;
}

interface Category {
  id: string;
  name: string;
  slug: string;
  description?: string;
  icon?: string; // 分类图标
  order: number; // 排序
}

interface Instructor {
  id: string;
  name: string;
  avatar: string;
  bio: string; // 简介
  title: string; // 职位/头衔
  experience: string; // 教学经验
  email: string;
  socialLinks?: {
    linkedin?: string;
    twitter?: string;
    github?: string;
  };
}

interface Chapter {
  id: string;
  title: string;
  duration: number; // 章节时长（分钟）
  topics: string[]; // 章节主题列表
  order: number; // 章节顺序
}

interface FAQ {
  id: string;
  question: string;
  answer: string;
  order: number;
}
```

     - 难度等级（单选：初级/中级/高级，必填）

   - **课程内容** Tab：
     - 学习目标（多个输入框，可动态增删）

     - 适合人群（文本框）

   - **讲师信息** Tab：
     - 选择讲师（下拉选择，必填）
     - 如果讲师不存在，提供"创建新讲师"快捷入口

## 四、技术架构建议

### 4.1 前端技术栈（轻量化方案）

#### 4.1.1 核心框架
- **Next.js 14** (App Router)
  - 集成前端和后端，一个项目搞定
  - 支持 SSR 和 SSG，提升 SEO
  - 文件系统路由，简单直观
  - 内置 API Routes，无需独立后端

#### 4.1.2 样式方案
- **Tailwind CSS**：原子化 CSS，轻量快速
  - 无需额外的 UI 组件库，直接用 Tailwind 构建
  - 或选配轻量组件库（可选）：
    - daisyUI：基于 Tailwind 的组件库
    - Headless UI：无样式组件，完全控制样式

#### 4.1.3 状态管理
- **React Server Components**（默认，Next.js 内置）
- **React Context**（客户端状态，React 原生）
- 不需要额外的状态管理库（Zustand/Redux）

#### 4.1.4 表单处理
- **React Hook Form**：轻量级表单库
- **Zod**：TypeScript 优先的验证库
- 两者结合，类型安全的表单处理

#### 4.1.5 数据获取
- **Server Actions**（Next.js 14 推荐方式）
- **Fetch API**（原生，无需额外库）
- 不需要 SWR 或 TanStack Query（保持简单）

#### 4.1.6 其他工具
- **富文本编辑器**：
  - Tiptap（轻量，推荐）
  - 或 React-Quill（更简单）
- **图标**：
  - lucide-react（轻量图标库）
  - 或直接使用 SVG
- **日期处理**：
  - date-fns（轻量，按需引入）
  - 或原生 JavaScript Date

### 4.2 后端技术栈（本地部署版本）
- **API 层**：
  - Next.js API Routes / Server Actions
  - 集成在 Next.js 中，无需独立服务

- **数据库**：
  - **SQLite**：轻量级文件数据库，无需安装独立数据库服务
  - **Prisma ORM**：类型安全的数据库访问，支持 SQLite
  - 数据库文件存储在项目目录：`prisma/dev.db`

- **认证**：
  - 简化方案：使用 `iron-session` 或 `next-auth` 的简化配置
  - Session 存储在 cookie 中
  - 无需外部 OAuth 服务

- **文件存储**：
  - **本地文件系统**：存储在 `public/uploads/` 目录
  - 子目录结构：
    - `public/uploads/courses/` - 课程封面
    - `public/uploads/instructors/` - 讲师头像
    - `public/uploads/certificates/` - 证书图片
    - `public/uploads/banners/` - Banner 图片

### 4.3 本地开发和部署方案

#### 4.3.1 开发环境要求
- **Node.js**：v18.17+ 或 v20.0+
- **包管理器**：npm / yarn / pnpm（推荐 pnpm）
- **操作系统**：macOS / Linux / Windows
- **编辑器**：VS Code（推荐）

#### 4.3.2 本地运行方式
```bash
# 1. 安装依赖
pnpm install

# 2. 初始化数据库
pnpm prisma migrate dev
pnpm prisma db seed  # 可选：填充初始数据

# 3. 启动开发服务器
pnpm dev

# 访问地址
# 前台：http://localhost:3000
# 后台：http://localhost:3000/admin
```

#### 4.3.3 生产部署（本地服务器）
```bash
# 1. 构建生产版本
pnpm build

# 2. 启动生产服务器
pnpm start

# 或使用 PM2 保持进程运行
pm2 start npm --name "courseai" -- start
```

#### 4.3.4 数据备份
```bash
# 备份 SQLite 数据库
cp prisma/dev.db prisma/backups/dev.db.backup

# 备份上传的文件
tar -czf uploads-backup.tar.gz public/uploads/
```

#### 4.3.5 本地部署的优势

1. **零云服务成本**：
   - 无需购买云数据库服务
   - 无需购买云存储服务
   - 无需购买域名和 SSL 证书（开发阶段）

2. **极简运维**：
   - 一个命令启动：`pnpm dev`
   - 无需配置复杂的环境变量
   - 数据库和文件都在本地，方便查看和管理

3. **完全可控**：
   - 所有数据都在本地，隐私安全
   - 可以随时备份和迁移
   - 不受云服务商限制

4. **快速开发**：
   - 修改即生效，无需部署等待
   - 可以使用 Prisma Studio 直接查看和编辑数据
   - 调试方便，错误信息清晰

#### 4.3.6 性能优化建议

1. **SQLite 优化**：
   ```sql
   -- 启用 WAL 模式提升性能
   PRAGMA journal_mode=WAL;
   
   -- 设置缓存大小
   PRAGMA cache_size=10000;
   ```

2. **Next.js 优化**：
   - 使用 Image 组件优化图片加载
   - 启用 gzip 压缩
   - 使用动态导入减少首屏加载

3. **本地网络访问**（局域网内其他设备访问）：
   ```bash
   # 启动时绑定到所有网络接口
   pnpm dev -- -H 0.0.0.0
   
   # 然后可以通过局域网 IP 访问
   # 例如：http://192.168.1.100:3000
   ```

#### 4.3.7 注意事项

1. **文件大小限制**：
   - 建议单个上传文件不超过 10MB
   - 可在代码中配置限制
   
2. **数据库并发**：
   - SQLite 适合中小规模应用
   - 如果未来需要高并发，可迁移到 PostgreSQL（Prisma 支持无缝切换）

3. **定期备份**：
   - 建议每天备份一次数据库文件
   - 可以写个简单的 cron job 自动备份

   ```bash
   node --version  # 应该显示 v20.x.x
   npm --version   # 应该显示 10.x.x
   ```

```bash
npm install -g pnpm
pnpm --version  # 验证安装
```

#### 第三步：创建项目
```bash
# 克隆或创建项目
mkdir courseai
cd courseai

# 初始化 Next.js 项目
pnpm create next-app@latest . --typescript --tailwind --app
```

#### 第四步：安装依赖
```bash
# 安装核心依赖
pnpm add @prisma/client react-hook-form zod iron-session bcryptjs
pnpm add -D prisma @types/bcryptjs

# 初始化 Prisma
pnpm prisma init --datasource-provider sqlite
```

#### 第五步：配置数据库
1. 编辑 `prisma/schema.prisma` 定义数据模型
2. 创建迁移：
   ```bash
   pnpm prisma migrate dev --name init
   ```
3. 查看数据库：
   ```bash
   pnpm prisma studio
   ```

#### 第六步：启动开发服务器
```bash
pnpm dev

# 访问 http://localhost:3000
```

## 常见问题 FAQ

**Q1: 为什么选择 SQLite 而不是 PostgreSQL？**  
A: SQLite 是轻量级文件数据库，无需安装独立服务，非常适合本地开发和中小规模应用。而且 Prisma 支持无缝切换，未来如需要可以轻松迁移到 PostgreSQL。

**Q2: 项目能支持多少用户同时访问？**  
A: 本地部署的 SQLite 版本适合几十到几百个并发用户。如果需要更高并发，建议迁移到 PostgreSQL。

**Q3: 如何在局域网内让其他设备访问？**  
A: 使用 `pnpm dev -- -H 0.0.0.0` 启动，然后通过你的 Mac 的局域网 IP 访问，例如 `http://192.168.1.100:3000`。

**Q4: 上传的文件存储在哪里？**  
A: 存储在 `public/uploads/` 目录下，建议定期备份这个目录。

**Q5: 如何备份数据？**  
A: SQLite 数据库就是一个文件（`prisma/dev.db`），直接复制即可。上传的文件在 `public/uploads/`，也可以直接复制或打包备份。

**Q6: 能否部署到生产环境？**  
A: 可以，但建议：
- 使用反向代理（如 Nginx）
- 配置 SSL 证书
- 启用 gzip 压缩
- 设置适当的文件权限
- 定期自动备份

**Q7: Mac 电脑配置要求？**  
A: 最低要求：
- macOS 10.15 或更高
- 4GB 内存（8GB 推荐）
- 2GB 可用磁盘空间
- 任何能运行 Node.js 的处理器
