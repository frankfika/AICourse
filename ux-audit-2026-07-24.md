## 1. 死功能 (Dead UI)

- **`router.tsx:169` `/__design-system` 在生产环境未守卫** — `DesignSystemPage` 没包 `import.meta.env.DEV` 三元 (对比 `ErrorDemoPage` line 173-180 有守卫), 投资人进 `https://.../__design-system` 直接看到内部色板/字号 token 页面, 注释自承"临时挂载,后续 worktree 跑完移除"。**P0**
- `features/admin/AdminCoursesPage.tsx:1500-1515` `ResourcesTab` 函数保留 (注释: "兼容 router 引用, 改返回 EmptyState 避免误点"), 整个 admin scope 外不构成用户故事, 仅记.
- `features/profile/ProfilePage.tsx:12-14` 全文件 14 行, 仅 `<Navigate to="/dashboard" replace />` 薄壳, 注释说"如果 Frank 决定彻底删除 /profile,直接 delete 本文件 + 移除 router.tsx import 即可" — 死代码语义层, 但 router 仍在引用, 不算真死.

- **`features/dashboard/DashboardPage.tsx:763-790` 三个 `useQuery` 全没解构 `isError`** — `coursesQuery / courseQuery / progressQuery` 都 `retry: 0`, 网络挂 / 401 / 后端挂 → 走 line 870 `if (!course)` 分支, 显示"还没有可学习的课程" EmptyState, **错把错误显示成空数据**; line 16 注释自承"失败 / 401 / 网络错 → 渲染 EmptyState", 但实际没 `isError` 分支, **注释与代码矛盾**.
- **`features/dashboard/DashboardPage.tsx:814` `completeLessonMutation` 只 `onSuccess` 无 `onError`** — 标记课时失败静默, 用户连个 toast 都没.
- **`features/dashboard/DashboardLayout.tsx:45-82` 4 个 useQuery 全无 `isError`** — `courseFromUrl / myEnrollments / courseProgress / userPoints` 任一挂, 顶栏静默显示 "—", 用户不知道是没数据还是网络挂.
- **`features/dashboard/certificates/VerifyCertificatePage.tsx:24-28` `useQuery` 无 `isError`** — 公开证书验证, 网络挂 → 永远 `data === undefined`, line 41 `valid = data?.valid === true` 为 false, **错误证书永远显示"未找到该证书编号对应的记录"**, 第三方验证场景致命.
- **`features/hackathons/HackathonDetailPage.tsx:34-44` useQuery 无 `isError`** — line 65 `if (!hackathon)` 走 404 文本, 错误 / 404 / 网络挂三者无法区分.
- **`features/degrees/DegreeListPage.tsx:12-18` useQuery 无 `isError` 也无 `EmptyState`** — line 67-153 `else` 分支直接 `degrees?.map`, 若 `degrees` undefined 直接 crash 空白, 后端返 `[]` 时 hero 后一片空白, 没占位.
- **`features/dashboard/orders/OrderDetailPage.tsx:68-100` useQuery 无 `isError`** — line 113 `if (!order)` 显示"订单不存在", 错误与不存在混淆, 无重试.
- **`features/dashboard/certificates/CertificateDetailPage.tsx:52-80` useQuery 无 `isError`** — line 93 同样把错误显示成"证书不存在".
- `features/courses/CourseListPage.tsx:629` / `features/courses/CourseDetailPage.tsx:204` / `features/degrees/DegreeDetailPage.tsx:103` / `features/dashboard/notifications/NotificationsPage.tsx:188` / `features/dashboard/orders/OrdersPage.tsx:187` / `features/dashboard/certificates/CertificatesPage.tsx:145` — 6 个 page 正确接 `QueryErrorState`, 对照上面 6 个没接的.

## 3. 交互一致性

- **`features/auth/BindingsPage.tsx:164` 用 `window.confirm` 二次确认** — 全项目其他危险操作 (OrdersPage 237/248, NotificationsPage 284) 都走 `<ConfirmDialog>`, 此处走 native `confirm()` 弹原生框, 风格分裂 + 不可定制 i18n.
- **`features/enterprise/EnterprisePage.tsx:519-523` 表单错误用 inline `<div className="border-red-600 bg-red-50">`** — 其他 mutation 错误 (OrdersPage 110-134, LoginPage 93-94) 都走 `useToast()`, 公开页 EnterprisePage 走 inline 红条, 错误提示体系两条平行轨.
- `features/enterprise/EnterprisePage.tsx:187-200` 整个表单没用 `react-hook-form` + `zod`, 手撸 `useState<InquiryForm>` — 跟 `LoginPage` / `RegisterPage` / `ForgotPasswordPage` (都用 `useForm({ resolver: zodResolver })`) 风格分裂, 双提交/字段失焦校验都缺.
- `features/dashboard/DashboardPage.tsx:897-912` mobile 3-tab (`大纲/视频/AI 助教`) 仅在 dashboard 内部, dashboard 子页 (`/dashboard/orders /notifications /certificates /orders/:id /certificates/:id`) 没继承 mobile tab, 改用 DashboardLayout 顶栏, 但**这些子页本身也没适配 mobile 断点** (OrdersPage 499 行纯 desktop grid `grid-cols-3`, mobile 单列勉强, 但 tab 切换没接).

## VERDICT: **FAIL**

**P0 缺口 (3 项)**:
1. `router.tsx:169` `__design-system` 生产环境未守卫
